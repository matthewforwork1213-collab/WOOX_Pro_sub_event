## 2. 기능 — 영역별 노출 on/off (5개)

관리자는 아래 5개 항목을 각각 독립적으로 on/off한다(부분 제어 허용). 저장 즉시 유저 화면 신규 요청에 반영된다.

| # | 백오피스 항목 | 제어 대상(유저 프론트) | 저장 키 | OFF 시 유저 화면 |
|---|---|---|---|---|
| 1 | 출금완료 · WOOX Pro 가상 피드백 안내 | 기능 1 (S1) | `s1Feedback` | 가상 피드백 미노출 → 기존 광고 로직 |
| 2 | 출금완료 · 빗썸 트레블룰 배너 | 기능 3 배너 #2 (S1) | `s1Banner` | 배너 미노출 |
| 3 | 캐시백 프리뷰 · WOOX Pro 비교 안내 | 기능 2 (S2) | `s2Compare` | 비교 카드 미삽입(결과 화면만) |
| 4 | 캐시백 프리뷰 · 빗썸 트레블룰 배너 | 기능 3 배너 #1 (S2) | `s2Banner` | 배너 미노출 |
| 5 | 로그인 3페이지 · 빗썸 트레블룰 배너 (일괄) | 기능 3 배너 #3·#4·#5 | `loginBanners` | PC 로그인·모바일 로그인전·마이페이지 세 위치 일괄 미노출 |

- **독립 제어**: 특정 영역만 OFF 가능. 기존 "3기능 일괄 롤백·부분 종료 없음"은 폐지.
- **기간 통제**: 별도 종료 일정 자동화 없이 관리자가 on/off로 시작·종료를 통제한다(D+30 자동 종료 없음).
- **WOOX Pro 포함 출금 이벤트 분기(F-002)** 는 본 5개 토글 대상이 아니다(상시 동작).

## 3. 권한 · 감사 · 데이터

| 항목 | 내용 |
|---|---|
| 접근 권한 | 기존 어드민 권한 체계 재사용. 프로모션 운영 권한 보유자만 조회/변경 가능 (⚠️ OI-B1: 세부 롤 범위 확정 필요) |
| 감사 로그 | 각 토글 변경 시 **변경자·시각·이전값·이후값** 기록 (⚠️ OI-B2: 감사 저장소·보존 기간 확정 필요) |
| 저장 데이터 | 5개 불리언 플래그. 서버 영속 저장(프로토타입은 localStorage로 시뮬레이션). 기본값=전부 ON |
| 반영 시점 | 저장 후 신규 요청 기준 즉시(수 분 이내). 게이트 조회 결과 장기 캐시 금지 (NFR-004) |
| 장애 시 | 설정 조회 실패 시 유저 프론트는 안전하게 **전부 OFF**로 폴백(넛지·배너 미노출) |

## 4. 유저 프론트와의 관계

- 유저 프론트는 `GET /api/promo/status`(API-001)로 5개 플래그를 읽어 각 영역 렌더를 결정한다(기능명세서_FE/BE F-005).
- 백오피스는 `GET/PUT /api/admin/promo/visibility`(API스펙 백오피스 API)로 5개 플래그를 조회·저장한다.
- 즉 **백오피스가 쓰고(write), 유저 프론트가 읽는다(read).** 동일한 5개 플래그를 공유.

## 5. 범위 외 (Out of Scope)

- 할인율·페이백율·거래소 온보딩·이벤트 등록/종료 등 기존 어드민 값 (신규 개발 아님, 읽기만)
- 트레블룰 배너 클릭 목적지 URL 관리(OI-06, 기존 온보딩 등록 값 사용)
- 어드민 "테더맥스 적용" 필드 이중곱 버그(OI-10, 별도 엔지니어링 트랙)
