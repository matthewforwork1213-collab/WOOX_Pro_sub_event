# PRD: WOOX Pro 온보딩 기념 30일 집중 프로모션 기능 정의

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **목적** | WOOX Pro 거래소 신규 온보딩 직후 30일간 테더맥스 플랫폼 내 트래픽과 UI 넛지(Nudge)를 활용해 WOOX Pro로의 유저 전환율 및 가입률을 극대화 |
| **핵심 전략** | 유저의 **'손실 회피 편향(Loss Aversion)'** 을 자극하는 비교 수치 제시 및 적절한 타이밍의 UI 강제 환기 (딤 처리 및 모달) |
| **프로모션 기간** | WOOX Pro 온보딩 시점으로부터 **30일간** (이후 기존 로직으로 롤백 또는 상시화 검토) |

## 2. 세부 기능 요구사항 (Functional Requirements)

### 기능 1: 출금 완료 후 WOOX Pro 프로모션 분기 노출

#### 기존 로직

유저가 (여러 거래소 동시 출금을 포함하여) 출금을 완료하면, 화면 내 우선순위에 따라 현재 진행 중인 거래소 이벤트 중 임의의 1개를 광고 형태로 노출.
> 예: 토스 송금 후 보험 광고 형태

#### 변경 및 신규 로직 (30일 한정)

**케이스 A: WOOX Pro가 포함되지 않은 출금 건**

- 기존의 일반 이벤트 광고 대신, **"만약 WOOX Pro를 사용했더라면 이번에 얼마를 더 출금할 수 있었는지"** 계산된 가상 피드백 문구를 광고 영역에 노출
- **UI/UX:** 시각적으로 유저가 이득을 놓쳤다는 느낌(기회비용)을 주는 카피라이팅 적용

**케이스 B: WOOX Pro가 포함된 출금 건**

| 조건 | 노출 내용 |
|------|-----------|
| WOOX Pro 자체 진행 이벤트가 **있을** 시 | **WOOX Pro 이벤트**를 최우선 노출 |
| WOOX Pro 자체 진행 이벤트가 **없을** 시 | **테더맥스 타입(기본) 이벤트**를 노출 |

---

### 기능 2: 캐시백 프리뷰(페이백 미리보기) 결과 내 WOOX Pro 비교 모달 강제 노출

#### 진입 맥락

유저가 '페이백 미리보기'에서 다음 조건을 입력한 후 **[결과 페이지]에 진입한 시점**에 작동:
- 거래소
- 자산 규모
- 레버리지
- 거래 타입
- 거래 빈도

#### 작동 로직

**케이스 A: 선택한 거래소가 WOOX Pro가 아닌 경우 (예: BitMart 등)**

1. 결과 페이지 로드와 동시에 배경 화면을 **딤(Dim) 처리**하여 흐리게 만듦
2. 화면 중앙에 **[WOOX Pro 비교 모달 팝업]** 을 강제로 띄움

**모달 내부 포함 콘텐츠:**

| 요소 | 상세 내용 |
|------|-----------|
| **손실 금액 인지** | "만약 WOOX Pro를 사용했다면 현재 이용하려는 거래소(예: BitMart)에 비해 아낄 수 있었던 예상 금액(USDT) 및 아낄 수 있는 퍼센테이지 포인트(%)"를 시각적으로 명확히 대비하여 표시 |
| **CTA 버튼 (Primary)** | WOOX Pro 계산 결과 상세 보기 또는 WOOX Pro 거래소 상세/가입 페이지로 바로 이동하는 버튼 (테더맥스 브랜드 컬러 적용, 강조) |
| **닫기 버튼 (Secondary)** | 우측 상단 'X' 버튼 배치. *(선택 사항: 하단에 "아니요, 그냥 기존 거래소 결과 볼게요" 등의 마이너 텍스트 링크 추가 검토)* |

**케이스 B: 선택한 거래소가 WOOX Pro인 경우**

- 딤 처리 및 비교 모달을 **절대 표시하지 않음**
- 즉시 WOOX Pro의 정상 페이백 결과 화면만 노출

## 2-A. 기간 한정 추가분 상세 로직 (a~d)

§2의 기능 1·2 중 **프로모션 기간에만 적용**되는 분기 (a)~(d)의 구현 상세이다.
(a)=기능 1 WOOX Pro 미포함, (b)=기능 1 WOOX Pro 포함, (c)=기능 2 타 거래소, (d)=기능 2 WOOX Pro.

### 2-A-1. 절약액 계산 모델 (공통)

유저가 받는 혜택은 거래량이 아니라 **커미션 × 페이백율**로 산정한다.

```
유저 수령 혜택(거래소 E) = 커미션(E) × 페이백율(E)
  · 커미션(E)   = 수수료(E) × 커미션율(E)   (파트너 계정이 받는 분)
  · 페이백율(E) = 커미션 중 유저에게 환급하는 비율

추가 이득(= 놓친 금액) = WOOX Pro 혜택 − 현재 거래소 혜택   (양수일 때만 노출)
  · WOOX Pro 혜택 = 커미션(WOOX Pro) × 80%
```

| WOOX Pro 파라미터 | 값 | 성격 |
|------|------|------|
| 커미션율 | 90% (테더맥스 내 최고 수준) | 내부 전용 (비노출) |
| 페이백율(캐시백) | 80% | 내부 전용 (비노출) |
| 유저 실효 페이백 | 수수료의 72% (90% × 80%) | 내부 산식 |

- **데이터 소스**: 커미션율·페이백율·유저별 실제 발생 커미션 모두 테더맥스 내부 보유. 외부 의존 없음.
- **노출 원칙**: 위 내부 수치는 유저 화면에 절대 표시하지 않고 **결과값(절약 금액 USDT, %p)만** 노출한다. (CLAUDE.md 핵심 비즈니스 규칙 6 참조)

### 2-A-2. (a) WOOX Pro 미포함 출금 → 가상 피드백

| 우선순위 | 노출 콘텐츠 | 조건 |
|------|------|------|
| 1 | 가상 피드백 ("WOOX Pro였다면 +X USDT") | 절약액 ≥ 임계값 & 계산 가능 |
| 2 | 테더맥스 WOOX Pro 온보딩 이벤트 | 1번 불가 시 & 이벤트 활성 |
| 3 | 기존 하우스/타 거래소 이벤트 (base 폴백) | 1·2 모두 불가 시 |

**복수 거래소 동시 출금**: 거래소별 추가 이득을 각각 계산 → **양수인 거래소만 합산** → 단일 총액으로 표시. 역효과(음수) 거래소는 합산에서 제외. 하단에 "N개 거래소·일반 등급 기준" 명시.

### 2-A-3. (b) WOOX Pro 포함 출금 → 이벤트 분기

| 우선순위 | 노출 콘텐츠 | 조건 |
|------|------|------|
| 1 | WOOX Pro 자체 이벤트 | WOOX Pro 이벤트 활성 |
| 2 | 테더맥스 WOOX Pro 온보딩 이벤트 (= "테더맥스 타입 이벤트") | 1번 없을 때 & 활성 |
| 3 | 테더맥스 일반 하우스 광고 | 1·2 모두 없을 때 |

- WOOX Pro 이벤트는 **절대 우선** (동시 출금에 타 거래소 이벤트가 있어도 WOOX Pro가 우선).
- 이벤트 유무 판정은 **어드민 등록 기반**. 경쟁 거래소 이벤트는 노출하지 않음(파트너십 보호).

### 2-A-4. (c) 타 거래소 선택 → WOOX Pro 비교 모달

**노출 조건 (AND)**: 타 거래소 선택 · 유저 WOOX Pro 미연동 · 정확 비교 가능 · 절약액 > 0

- 비교식: 위 2-A-1 모델 (양쪽 페이백 적용 후 실질 혜택 비교).
- 이미 WOOX Pro 연동 유저: 미노출.
- WOOX Pro 미지원 조건(레버리지/상품): 미노출 (부정확 비교 차단).
- 세션 정의: 웹뷰/탭 단위, 30분 무활동 시 만료. **세션당 1회** + "하루 보지 않기" 옵션 제공.

### 2-A-5. (d) WOOX Pro 선택 → 모달 미표시

딤·비교 모달 미표시, 정상 페이백 결과 화면만 노출. (c) 노출 조건 미충족 케이스 전부 포함.

### 2-A-6. 프로모션 기간 기준

- "온보딩 시점"은 **글로벌 고정일**(배포일 또는 파트너 지정일)부터 전 유저 동일하게 D+30 적용.
- 기능 1·기능 2는 **독립 Feature Flag**로 제어하여 기능별 개별 종료 가능 (§3-3 참조).

## 3. 예외 처리 및 데이터 정책 (Exception & Data Policy)

### 3-1. 수수료 비교 계산의 정확성 및 신뢰성 확보

- 유저가 입력한 조건에서 **타 거래소의 페이백 금액이 WOOX Pro보다 더 크게 계산되는 특이 케이스** (예: 타 거래소 최고 VIP 등급 적용 등)가 발생할 경우, **출금 후 광고 및 프리뷰 모달을 절대 노출하지 않음** (역효과 방지)
- 비교 기준은 **'일반 유저/기본 등급'** 혹은 명확한 산정 기준을 모달 하단에 아주 작은 텍스트로 명시하여 신뢰도 확보

### 3-2. 유저 피로도 방어 로직 (UX 방어 로직)

**문제 상황:**
캐시백 프리뷰에서 유저가 조건을 변경해가며 단시간 내 여러 번(예: 3회 이상) 계산을 반복할 때마다 모달이 반복 노출될 경우 이탈률이 높아질 수 있음.

**해결 방안:**

| 방안 | 내용 |
|------|------|
| **방안 1** | 비교 모달 노출을 **'세션당 1회'** 또는 **'하루 동안 보지 않기'** 옵션을 모달 내 제공 |
| **방안 2** | 팝업 형태가 과하다고 판단될 경우, 화면을 가리는 모달 대신 결과 카드 위/아래에 고정되는 **'인라인 배너'** 혹은 **'바텀 시트'** 형태로 전환 가능성 검토 (A/B 테스트 권장) |

### 3-3. 프로모션 조기 종료 및 어드민 제어

프로모션은 자동 만료(D+30) 외에도 어드민을 통해 30일 이전에 조기 종료할 수 있으며, **종료 단위는 기능별(이벤트별)로 독립적**이다.

| 항목 | 내용 |
|------|------|
| **자동 종료** | WOOX Pro 온보딩 시점 기준 D+30 도달 시 자동으로 평상시 로직으로 롤백 |
| **어드민 조기 종료** | D+30 이전에도 어드민이 수동으로 종료 가능 |
| **종료 단위** | 기능 1(출금 후 분기)·기능 2(비교 모달)는 **각각 독립된 종료 스위치**를 보유 |
| **부분 종료** | 어드민이 2개 기능 중 **하나만 종료해도 해당 기능만 종료**되며, 나머지 기능은 정상 동작 유지 |
| **종료 트리거** | (a) D+30 자동 만료 또는 (b) 어드민 수동 종료 중 **먼저 도달하는 시점** 기준 |

**종료 시 기능별 롤백 거동:**

| 종료 대상 | 롤백 후 거동 |
|------|------|
| **기능 1 종료** | 출금 완료 후 광고를 기존 "진행 중 거래소 이벤트 중 임의 1개 노출" 로직으로 복귀. 가상 피드백 미노출. |
| **기능 2 종료** | 캐시백 프리뷰 결과에서 딤 처리·비교 모달 미표시. 기존 정상 결과 화면만 노출. |

**구현 요구사항:**

- 기능 1·기능 2는 각각 **독립적인 Feature Flag**로 제어하여 즉시 ON/OFF가 가능해야 한다.
- 한 기능의 종료가 다른 기능의 동작에 영향을 주지 않아야 한다 (완전 분리).
- 종료 시점에 이미 노출 중인 화면(예: 이미 떠 있는 모달)의 처리 방식, 어드민 종료 권한 범위, 종료 후 재개(재활성화) 가능 여부는 추후 정의한다.

## 4. 플로우 요약

```
[출금 완료 후]
│
├── WOOX Pro 미포함 출금
│   └── 광고 영역: "WOOX Pro 사용 시 +OOO USDT 수령 가능했음" 가상 피드백 노출
│
└── WOOX Pro 포함 출금
    ├── WOOX Pro 자체 이벤트 있음 → WOOX Pro 이벤트 최우선 노출
    └── WOOX Pro 자체 이벤트 없음 → 테더맥스 기본 이벤트 노출

[페이백 미리보기 결과 페이지 진입 후]
│
├── 선택 거래소 ≠ WOOX Pro
│   ├── 배경 Dim 처리
│   └── WOOX Pro 비교 모달 강제 노출
│       ├── 손실 금액 비교 (USDT / %)
│       ├── CTA: WOOX Pro 가입/상세 이동 버튼
│       └── 닫기: X 버튼 (Secondary)
│
└── 선택 거래소 = WOOX Pro
    └── Dim 없이 정상 페이백 결과 화면만 노출

[프로모션 종료 판정 — 각 기능 진입 전 공통 선행 체크]
│
├── 기능 1 Flag = OFF (D+30 만료 또는 어드민 종료)
│   └── 기능 1 → 기존 "거래소 이벤트 임의 1개" 로직으로 동작
│
└── 기능 2 Flag = OFF (D+30 만료 또는 어드민 종료)
    └── 기능 2 → 비교 모달 없이 기존 결과 화면만 노출
    (※ 두 Flag는 독립 — 한쪽 종료가 다른 쪽에 영향 없음)
```
