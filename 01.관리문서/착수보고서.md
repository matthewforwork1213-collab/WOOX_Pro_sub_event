## 1. 프로젝트 개요

TetherMax 플랫폼 내 유저 행동 접점(출금 완료 화면, 캐시백 프리뷰 결과 페이지)에 손실 회피 편향(Loss Aversion) 기반 UI 넛지를 삽입하여, WOOX Pro 신규 온보딩 시점으로부터 30일간 유저의 WOOX Pro 전환율과 가입률을 집중적으로 극대화한다.

| 항목 | 내용 |
|---|---|
| 파트너사 | TetherMax × WOOX Pro |
| 프로모션 기간 | WOOX Pro 거래소 온보딩(런칭) 시점으로부터 30일간 (이후 기존 로직 롤백, 상시화 미적용) |
| 대상 플랫폼 | Web PC · Web MO · App(iOS/Android 웹뷰) |
| 개발 방식 | 웹뷰 우선, 기존 TetherMax 플랫폼에 기능 추가 |
| 핵심 기능 | 기능 1(출금 완료 후 가상 피드백), 기능 2(캐시백 프리뷰 인라인 비교 카드), 기능 3(WOOX Pro × 빗썸 트레블룰 연동 배너) |

## 2. 프로젝트 목표

| 구분 | 목표 | 측정 지표 |
|---|---|---|
| 단기 (프로모션 30일 기간 내) | 출금 완료·캐시백 프리뷰 접점에서 WOOX Pro 전환 유도 | 기능 1 CTR 4% 이상, 기능 2 비교 카드 CTR 7% 이상, WOOX Pro 가입(KYC) 완료율 30% 이상 (서비스기획서 §9 참조) |
| 단기 | 역효과·과오 노출 0건 | 타 거래소가 실제로 유리한 케이스에서 넛지 노출 0건 (§3-1 역효과 방지 로직 QA 검증) |
| 중기 (프로모션 종료 후) | 프로모션 성과 데이터 확보 → 차기 파트너 프로모션 설계 근거 마련 | D+30 시점 WOOX Pro 신규 KYC 완료 건수, CTR·이탈률 추이 리포트화 (Go/No-Go 기준은 상시화가 아닌 차기 캠페인 참고용) |

## 3. 프로젝트 범위

### 3.1 범위 내 (In Scope)

- 기능 1: 출금 완료 후 WOOX Pro 프로모션 분기 노출(가상 피드백/이벤트 분기)
- 기능 2: 캐시백 프리뷰 결과 페이지 내 WOOX Pro 비교 카드 인라인 노출
- 기능 3: WOOX Pro × 빗썸 트레블룰 연동 배너 (5개 위치)
- 위 3개 기능의 계산 로직, 역효과 방지 로직, 프로모션 활성/종료 판정 로직
- 대상 화면 S1(출금 완료)·S2(캐시백 프리뷰 결과)·S3(로그인 페이지, PC)·S4(로그인 전 메뉴, MO)·S5(마이페이지) — 화면 상세는 `02.기획문서/화면변경목록.md` 참조
- Web PC·Web MO 전체, App은 S1·S2에 한해 웹뷰로 대응

### 3.2 범위 외 (Out of Scope)

- 프로모션 조기 종료 기능 신규 개발 — 기존 2개 WOOX Pro 이벤트 어드민의 종료 버튼을 그대로 활용 (PRD §3-3)
- 캐시백 프리뷰 엔진의 "테더맥스 적용" 필드 cashbackRate 이중곱 버그 수정 — 별도 엔지니어링 트랙 (OI-10)
- WOOX Pro 자체 이벤트 운영·등록 기능(기존 어드민 기능 재사용, 신규 개발 아님)
- 프로모션 상시화(OI-07 종결 — 30일 한정 운영만 진행, 상시 전환 없음)
- S3~S5의 앱 네이티브 화면 대응(로그인 페이지·로그인 전 메뉴·마이페이지는 앱에서 네이티브 영역이라 본 프로젝트 범위 아님)
- WOOX Pro 외 타 거래소 온보딩 프로모션

## 5. 팀 구성

| 역할 | 인원 | 담당 영역 |
|---|---|---|
| PM | 미정 | 기획 총괄, 산출물 관리, C레벨 의사결정(예: OI-08 표시 방식 확정) 조율 |
| 프론트엔드 개발 | 미정 | S1~S5 화면 구현, 인라인 비교 카드·가상 피드백 카드·트레블룰 배너 컴포넌트 (§6 FE 작업) |

| 디자이너 | 미정 | S1~S5 화면 시안, 트레블룰 배너 로고 하드코딩 에셋, 비교 카드/가상 피드백 카드 UI |

> 실제 배정 인원은 조직 내 협의 후 확정한다. 인원 미확정이 산출물 작성 자체를 막지는 않는다.
