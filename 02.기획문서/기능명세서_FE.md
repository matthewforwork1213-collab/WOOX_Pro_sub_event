## 기능 상세

### F-001. 출금 완료 후 가상 피드백 카드 (WOOX Pro 미포함)

| 항목 | 내용 |
|---|---|
| 설명 | 출금 완료 화면(S1)에서 WOOX Pro가 포함되지 않은 출금 건에 대해, 기존 광고 영역을 가상 피드백 카드로 대체 노출한다 |
| 입력 | 화면 진입 이벤트(출금 완료), 출금 UID 목록(복수 가능) — 유저 별도 입력 없음 |
| 처리 | 1) 화면 진입 시 BE에 출금 UID 목록으로 가상 피드백 데이터 요청 2) 응답의 `노출여부`가 true면 가상 피드백 카드 렌더, false면 F-002 또는 base 이벤트 로직으로 폴백 3) **토탈 세이빙 기준 카피 템플릿**("OOO USDT 더 아낄 수 있었어요")으로 렌더(OI-08 확정) 4) UID 2개 이상이면 합산 총액 + "N개 거래소 기준" 캡션 렌더(개별 UID 나열 없음) |
| 출력 | 가상 피드백 카드(헤드라인, 절약 금액 USDT, %p, 서브 카피, CTA 버튼) |
| 예외 처리 | 응답 `노출여부=false`(역효과·계산불가·절약액<1USDT 처리 후 0 이하) 시 카드 미노출, F-002/base 로직으로 대체. API 실패/타임아웃 시 base 이벤트 로직으로 폴백(가상 피드백 노출 안 함). 배치 시차 안내 문구 등 시각 정보를 표시할 경우, **API가 내려주는 UTC 타임스탬프를 브라우저/기기의 로컬 타임존으로 변환**해 렌더링(NFR-007) — KST 하드코딩 금지 |
| 관련 화면 | S1 (출금 완료 화면) — PC·MO·App(웹뷰) |
| 관련 API | GET /api/promo/withdrawal-feedback (잠정, UID 목록 기준 조회) |

### F-002. 출금 완료 후 WOOX Pro 이벤트 분기 (WOOX Pro 포함)

| 항목 | 내용 |
|---|---|
| 설명 | WOOX Pro가 포함된 출금 건에 대해 (1)WOOX Pro 자체 이벤트 (2)테더맥스 WOOX Pro 온보딩 이벤트 (3)일반 하우스 광고 순으로 분기 노출한다 |
| 입력 | 화면 진입 이벤트(출금 완료), 출금 UID 목록 |
| 처리 | BE 응답의 `이벤트유형` 필드(`woox_event` / `onboarding_event` / `house_ad`)에 따라 대응하는 광고 컴포넌트를 렌더 |
| 출력 | 이벤트 유형별 광고 카드(WOOX Pro 자체 이벤트 배너 / 온보딩 이벤트 배너 / 일반 하우스 광고) |
| 예외 처리 | API 실패 시 기존 "진행 중 거래소 이벤트 임의 1개 노출" base 로직으로 폴백 |
| 관련 화면 | S1 (출금 완료 화면) — PC·MO·App(웹뷰) |
| 관련 API | GET /api/promo/withdrawal-feedback (F-001과 동일 엔드포인트, 응답 필드로 케이스 분기) |

### F-003. 캐시백 프리뷰 WOOX Pro 비교 카드 (인라인)

| 항목 | 내용 |
|---|---|
| 설명 | 캐시백 프리뷰 결과 페이지(S2)에서 타 거래소 선택 시, 기존 결과 카드 바로 아래에 WOOX Pro 비교 카드를 팝업·딤 없이 인라인 삽입한다 |
| 입력 | 유저가 프리뷰에서 입력한 조건(거래소, 자산 규모, 레버리지, Maker/Taker 비율, 거래 빈도), 카드 접기 토글 클릭 |
| 처리 | 1) 프리뷰 조건 입력값을 BE에 그대로 전달해 비교 결과 요청 2) 응답 `노출여부=true`면 결과 카드 하단에 비교 카드 렌더(펼침 상태 기본) 3) 카드 접기 버튼 클릭 시 로컬 상태로 펼침/접힘만 토글(재요청 없음) 4) **토탈 세이빙 기준 카피 템플릿**으로 렌더(OI-08 확정) |
| 출력 | 비교 카드(헤드라인, 절약 금액 USDT, %p, "일반 유저/기본 등급 기준" 캡션, CTA 버튼, 접기 토글) |
| 예외 처리 | 응답 `노출여부=false`(역효과·WOOX Pro 미지원 조건) 시 비교 카드를 렌더하지 않고 기존 결과 화면만 표시. 거래소=WOOX Pro 선택 시 애초에 API 미호출(비교 카드 개념 자체 불필요) |
| 관련 화면 | S2 (캐시백 프리뷰 결과 페이지) — PC·MO·App(웹뷰) |
| 관련 API | POST /api/promo/cashback-preview/compare (잠정, 프리뷰 입력 조건 기준 비교) |

### F-004. WOOX Pro × 빗썸 트레블룰 연동 배너 (5개 위치)

| 항목 | 내용 |
|---|---|
| 설명 | 프로모션 활성 기간 동안 5개 위치에 트레블룰 연동 정적 배너를 노출한다 |
| 입력 | 화면 진입 이벤트(5개 화면 각각) — 유저 별도 입력 없음, 클릭 시에도 별도 처리 없음(정적) |
| 처리 | 1) 화면 진입 시 프로모션 활성 여부(F-005) 조회 2) 활성 상태면 배너 컴포넌트 렌더 — 로고(`[빗썸 로고]``[WOOX Pro 로고]`, 디자인 하드코딩 이미지) + 문구(i18n 텍스트, 현재 로케일에 맞게 KO "트레블룰 연동" / EN "Travel Rule Integration" 등) 3) 로그인 전 화면(#3·#4)은 유저 세션과 무관하게 동일 로직 적용 |
| 출력 | 절제형 뱃지 스타일의 정적 배너(로고 2개 + 문구, 클릭 비활성) |
| 예외 처리 | 프로모션 비활성(F-005) 시 5개 위치 모두 배너 렌더 생략. App 환경에서는 S1·S2(출금 카드 하단, 캐시백 프리뷰 하단)에만 렌더하고 S3~S5(로그인 페이지·로그인 전 메뉴·마이페이지)는 네이티브 영역이므로 렌더 대상에서 제외 |
| 관련 화면 | S1(출금 결과 카드 하단 외부), S2(캐시백 프리뷰 결과 하단) — PC·MO·App(웹뷰) / S3(PC 로그인 페이지 상단), S4(MO 로그인 전 메뉴), S5(마이페이지 회원 ID 하단) — Web(PC/MO)만 |
| 관련 API | GET /api/promo/status (F-005와 동일 엔드포인트, 활성 여부만 사용) |

### F-005. 프로모션 활성 상태 반영 (화면 분기)

| 항목 | 내용 |
|---|---|
| 설명 | F-001~F-004가 공통으로 참조하는 프로모션 활성/종료 상태를 조회하고, 비활성 시 모든 화면을 base 로직으로 렌더한다 |
| 입력 | 화면 진입 이벤트 — 유저 별도 입력 없음 |
| 처리 | 1) 각 화면 진입 시(또는 공통 레이아웃 레벨에서 1회) 프로모션 활성 여부 API 조회 2) `활성=false`면 F-001·F-002는 base 이벤트 로직, F-003은 비교 카드 미노출, F-004는 배너 미노출로 즉시 분기 |
| 출력 | 없음(직접 렌더 요소는 없고 F-001~F-004의 렌더 여부를 결정하는 공통 상태값) |
| 예외 처리 | API 실패 시 안전한 기본값으로 **비활성** 처리(프로모션 넛지 미노출, 기존 로직 유지) — 장애 시에도 기존 서비스 경험을 해치지 않도록 함 |
| 관련 화면 | S1~S5 전체 (공통 레이어) |
| 관련 API | GET /api/promo/status (잠정) |
