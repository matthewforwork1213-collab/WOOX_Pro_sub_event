## 0. 문서 사용 안내 (FE/BE 구분 정의)

- **FE/BE는 "개발 영역" 구분이다**(프론트엔드 코드 / 백엔드 코드). 유저 프론트 vs 백오피스 구분이 아니다.
- 5개 기능(F-001~F-005)은 **전부 유저 프론트 기능**이며, 동일 기능을 화면 담당(FE)·계산 담당(BE)이 나눠 명세한다. 하나의 기능이 양쪽 문서에 모두 등장한다.
- **백오피스(어드민) 신규 화면 없음.** 기존 값 읽기·기존 버튼 재사용만 한다(상세는 `기능명세서_BE.md` §5).

## 1. 공통 렌더링 정책 (F-001~F-005 공통)

각 기능 상세(§2)는 이 절을 반복하지 않고 참조한다.

### 1.1 노출 제어 — 백오피스 영역별 on/off (2026-07-07 정책)

> **정책 변경(2026-07-07)**: 기존 "D+30 + 2개 WOOX Pro 이벤트" 공통 게이트 폐지 → **백오피스 영역별 on/off**로 제어. F-005가 5개 플래그를 제공. ⚠️ D+30 폐지는 OI-07 충돌·C레벨 재승인 대기.

- 각 영역은 대응 백오피스 플래그가 **OFF일 때** 아래처럼 폴백한다(영역별 독립).

| 영역 | 백오피스 플래그 | OFF 시 |
|---|---|---|
| F-001 가상 피드백 | `s1Feedback` | 카드 미노출 → **원 base 이벤트 로직(진행 중 거래소 이벤트 임의 1개)** |
| F-003 비교 카드 | `s2Compare` | 카드 미삽입 → 기존 결과 화면만 |
| F-004 배너 #2(S1) | `s1Banner` | 해당 배너 미노출 |
| F-004 배너 #1(S2) | `s2Banner` | 해당 배너 미노출 |
| F-004 배너 #3·#4·#5 | `loginBanners` | 세 위치 일괄 미노출 |

> **주의 — 두 경우를 구분한다.** 위는 **백오피스 OFF**(노출 제어)다. 반면 플래그 ON인데 개별 요청이 **자격 미달(`visible=false`: 역효과·계산불가)**이면 F-001은 **온보딩 이벤트 → 하우스 광고** 3단 폴백을 탄다(§2 F-001 처리, PRD §2-A-2). F-002(WOOX 포함 이벤트 분기)는 별도 노출 플래그 없이 상시 동작.

- **안전 기본값**: 설정 조회 실패 시 전부 **OFF**로 간주 — 장애 시에도 기존 서비스 경험을 해치지 않는다.

### 1.2 플랫폼 스코프 매트릭스

| 화면 | 위치 | Web PC | Web MO | App(웹뷰) |
|---|---|---|---|---|
| S1 | 출금 완료 화면 (F-001/F-002 + 배너 #2) | O | O | O |
| S2 | 캐시백 프리뷰 결과 (F-003 + 배너 #1) | O | O | O |
| S3 | PC 로그인 페이지 상단 (배너 #4) | O | - | X(네이티브) |
| S4 | MO 로그인 전 메뉴 (배너 #3) | - | O | X(네이티브) |

- **App 웹뷰 대상은 S1·S2뿐**이다. S3~S5는 앱에서 네이티브 영역이라 이번 범위 제외, Web(PC/MO)에서만 적용한다(NFR-006).

### 1.3 시간 표시 (로컬 타임 변환)

- API 응답의 타임스탬프는 UTC다. 화면에 시각(예: 배치 시차 안내 문구, NFR-005)을 표시할 경우 **반드시 사용자 기기의 로컬 타임존으로 변환**해 렌더링한다. **KST 고정 문구 노출 금지**(NFR-007).

### 1.4 표시 규칙 (금액·%p·내부수치)

- 금액은 **USDT 단독 표기**, 전체 다국어 지원(REQ-024).
- 증감은 **%가 아니라 %p(퍼센트포인트)** 로 표기한다.
- 절약액은 서버가 이미 1 USDT 미만 보정(1로) 완료해 내려주므로 FE는 받은 값을 그대로 표기한다(게이팅은 서버가 `visible`로 처리, `기능명세서_BE.md` §1.5).
- **내부 수치(할인율·페이백율·보정계수·커미션율·마진율)는 화면에 절대 표시하지 않는다.** 응답에도 포함되지 않는다(REQ-023).
- 모든 비교 카드·피드백 하단에 **"일반 유저/기본 등급 기준"** 소문자 캡션을 항상 표기한다(REQ-013).

### 1.5 국제화(i18n)

- 서버는 번역 텍스트가 아니라 **i18n 키**만 내려준다. 실제 문구는 FE 다국어 리소스에서 로케일별 렌더링한다(금액·%p 결과값은 서버 숫자 사용).
- 미지원 언어는 폴백 언어로 표시(NFR-003).

### 1.6 CTA

- Primary CTA는 **TetherMax 브랜드 컬러**를 적용한다(CLAUDE.md 규칙 5).
- CTA 목적지는 **WOOX Pro 거래소 상세 페이지**이며, URL은 **어드민 온보딩 등록 값**을 사용한다(OI-06, REQ-014).
