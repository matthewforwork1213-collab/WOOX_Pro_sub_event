## 1. 기능 요구사항 (Functional Requirements)

### 1.1 기능 1 — 출금 완료 후 WOOX Pro 프로모션 분기 노출

| ID | 구분 | 요구사항 | 우선순위 | 인수 기준 |
|---|---|---|---|---|
| REQ-001 | 기능 1 | WOOX Pro 미포함 출금 완료 시, 기존 이벤트 광고 대신 가상 피드백("WOOX Pro였다면 +X USDT")을 노출한다 | Must | 역효과 미해당 & 절약액 > 0 & 계산 가능 조건 충족 시 가상 피드백이 광고 영역에 노출됨 (1 USDT 미만 표시 보정은 REQ-003) |
| REQ-002 | 기능 1 | 가상 피드백 금액은 유저의 실제 페이백 금액(UID 테이블, 일 1회 KST 20:00 배치)을 역산하여 산출한다 | Must | `기준 수수료 = 실제 페이백 ÷ 페이백율` 계산 결과가 PRD §2-A-1 검증 예시와 일치 |
| REQ-003 | 기능 1 | 절약액이 1 USDT 미만이면 최소 표시값 1 USDT로 노출하고, 0 이하면 base 로직(기존 이벤트)으로 대체한다 | Must | 절약액 0.3 USDT 입력 시 "1 USDT" 노출, 절약액 0 이하 시 가상 피드백 미노출 |
| REQ-004 | 기능 1 | 복수 거래소(UID) 동시 출금 시 거래소별 추가 이득을 각각 계산하여 역효과 미해당·양수인 것만 합산한 단일 총액 하나로 표시한다 | Must | UID 5개 이상이어도 개별 나열 없이 총액 1개 + "N개 거래소 기준" 캡션만 노출 |
| REQ-005 | 기능 1 | WOOX Pro 포함 출금 시 (1)WOOX Pro 자체 이벤트 (2)테더맥스 WOOX Pro 온보딩 이벤트 (3)테더맥스 일반 하우스 광고 순으로 우선 노출한다 | Must | WOOX Pro 자체 이벤트가 어드민에 활성 상태면 타 거래소 이벤트가 있어도 WOOX Pro 이벤트가 최우선 노출됨 |
| REQ-006 | 기능 1 | 타 거래소 토탈세이빙율이 WOOX Pro와 같거나 높으면(역효과) 가상 피드백을 노출하지 않는다 | Must | 역효과 케이스 입력 시 가상 피드백 대신 base 이벤트 로직으로 폴백 |
| REQ-007 | 기능 1 | 가상 피드백 CTA 클릭 시 WOOX Pro 거래소 상세 페이지로 이동한다(URL은 어드민 온보딩 등록 값) | Must | CTA 클릭 시 어드민에 등록된 WOOX Pro 상세 URL로 정상 이동 |

### 1.2 기능 2 — 캐시백 프리뷰 결과 내 WOOX Pro 비교 카드 인라인 노출

| ID | 구분 | 요구사항 | 우선순위 | 인수 기준 |
|---|---|---|---|---|
| REQ-008 | 기능 2 | 타 거래소 선택 후 결과 페이지 진입 시, 기존 결과 카드 아래에 WOOX Pro 비교 카드를 팝업·딤 없이 인라인으로 삽입한다 | Must | 결과 페이지 로드 시 딤 처리·모달 없이 결과 카드 바로 아래 비교 카드가 렌더링됨 |
| REQ-009 | 기능 2 | 비교 카드의 예상 수치는 기존 캐시백 엔진의 Monthly Estimated Cashback 공식(balance·leverage·TIME·Maker/Taker 가중 수수료율·페이백율·보정계수 0.7)으로 산출한다 | Must | 동일 입력값으로 WOOX Pro·현재 거래소 각각 계산한 값이 PRD §2-A-1 공식과 일치 |
| REQ-010 | 기능 2 | 유저가 이미 WOOX Pro를 선택한 경우 비교 카드를 삽입하지 않는다 | Must | 거래소=WOOX Pro 선택 시 비교 카드 없이 기존 결과 화면만 노출 |
| REQ-011 | 기능 2 | 타 거래소의 토탈세이빙율(보정계수 반영)이 WOOX Pro와 같거나 높거나, WOOX Pro가 해당 조건(레버리지·상품)을 지원하지 않으면 비교 카드를 삽입하지 않는다 | Must | 역효과·미지원 조건 입력 시 비교 카드 미노출 |
| REQ-012 | 기능 2 | 비교 카드는 접기(collapse) 기능을 제공하며, 강제 닫기(X) UI는 두지 않는다 | Must | 카드 접기 버튼 클릭 시 카드가 접히고, 별도 닫기 버튼은 존재하지 않음 |
| REQ-013 | 기능 2 | 비교 카드 하단에 "일반 유저/기본 등급 기준"임을 소문자로 명시한다 | Must | 카드 하단에 해당 문구가 항상 표시됨 |
| REQ-014 | 기능 2 | 비교 카드 CTA 클릭 시 WOOX Pro 거래소 상세 페이지로 이동한다 | Must | CTA 클릭 시 정상 이동 확인 |
| REQ-015 | 기능 2 | (선택) 복수 UID 합산 총액에서 UID별 절약 내역을 펼쳐볼 수 있는 "내역 보기"를 제공할 수 있다 | Could | 내역 보기 클릭 시 UID별 절약액 목록이 펼쳐짐 (미구현 시에도 REQ-004 총액 표시는 유지) |

### 1.3 기능 3 — WOOX Pro × 빗썸 트레블룰 연동 배너

| ID | 구분 | 요구사항 | 우선순위 | 인수 기준 |
|---|---|---|---|---|
| REQ-016 | 기능 3 | 프로모션 활성 기간 동안 5개 위치(캐시백 프리뷰 결과 하단, 출금 결과 카드 하단 외부, MO 로그인 전 버튼 아래, PC 로그인 페이지 상단, 로그인 후 마이페이지 회원 ID 하단)에 트레블룰 연동 배너를 노출한다 | Must | 5개 위치 모두에서 배너가 정상 노출됨 (App은 S1·S2만 해당) |
| REQ-017 | 기능 3 | 배너는 `[빗썸 로고]` ⇄ `[WOOX Pro 로고]`(디자인 하드코딩, 로고 사이 좌우 화살표 ⇄) + 문구(KO "트레블룰 연동" / EN "Travel Rule Integration") 순서로 구성하며 전 언어 동일 순서를 유지한다 | Must | 언어 전환 시에도 로고-⇄-로고-문구 순서가 유지되고 문구만 번역됨 |
| REQ-018 | 기능 3 | 배너는 **클릭 시 WOOX Pro 거래소 상세 페이지로 이동**한다. CTA 목적지 URL은 어드민 온보딩 등록 값(OI-06). (기능 3 개정 2026-07-07 — 기존 "정적·클릭 이동 없음"에서 변경. ⚠️ 정적 정보 배너 전제였던 OI-09 법무 리스크 수용 재확인 필요) | Must | 배너 클릭(탭) 시 WOOX Pro 거래소 상세 페이지로 이동함 |
| REQ-019 | 기능 3 | 로그인 전 화면(#3·#4)의 배너 노출은 유저 세션과 무관하게 전역 프로모션 활성 여부만으로 판정한다 | Must | 비로그인 상태에서도 프로모션 활성 기간이면 배너가 노출됨 |

### 1.4 공통 요구사항 (프로모션 활성/종료·데이터·표시 규칙)

| ID | 구분 | 요구사항 | 우선순위 | 인수 기준 |
|---|---|---|---|---|
| REQ-020 | 공통 | 프로모션 활성 여부는 "(WOOX Pro 온보딩 시점 기준 D+30 이내) AND (2개 WOOX Pro 이벤트 모두 미종료)"로 판정한다 | Must | D+31 도달 또는 2개 이벤트 중 1개 종료 시 즉시 비활성으로 전환 |
| REQ-021 | 공통 | 프로모션 비활성 전환 시 기능 1·2·3을 동시에 일괄 base 로직으로 롤백한다 (부분 종료 없음) | Must | 종료 트리거 발생 시 세 기능이 같은 시점에 함께 base 상태로 복귀 |
| REQ-022 | 공통 | 프로모션 조기 종료는 기존 2개 WOOX Pro 이벤트 어드민 조회 화면의 종료 버튼을 그대로 사용하며, 신규 종료 UI는 개발하지 않는다 | Must | 신규 어드민 화면 없이 기존 종료 버튼 클릭만으로 REQ-021 동작이 트리거됨 |
| REQ-023 | 공통 | 유저 화면에는 결과값(절약 금액 USDT, %p)만 노출하고, 할인율·페이백율·보정계수·커미션율·마진율 등 내부 수치는 절대 노출하지 않는다 | Must | 응답 페이로드·화면 어디에도 내부 원자료 수치가 포함되지 않음 |
| REQ-024 | 공통 | 절약 금액은 USDT 단독 표시하며 전체 다국어를 지원하고, 증감 비율은 %(퍼센트)가 아닌 %p(퍼센트포인트) 단위로 표시한다. %p = 명목 토탈세이빙율(WOOX Pro) − 명목 토탈세이빙율(현재 거래소)로 정의하며(보정계수 미반영), 기능 1·2 공통 적용한다 (2026-07-03 확정, PRD §2-A-1) | Must | 모든 지원 언어에서 통화 표기가 USDT로 통일되고, 기능 1·2 어디서나 동일한 명목 토탈세이빙율 차이로 %p가 계산됨 |
| REQ-025 | 공통 | 가상 피드백/비교 카드의 표시 방식은 **토탈 세이빙 기준**으로 노출한다 (OI-08 확정, 2026-07-03 — 페이백률 기준은 폐기) | Must | 모든 가상 피드백·비교 카드 문구가 "OOO USDT 더 아낄 수 있었습니다" 형태의 토탈 세이빙 기준 프레이밍으로 노출됨 |

## 2. 비기능 요구사항 (Non-Functional Requirements)

| ID | 구분 | 요구사항 | 기준값 |
|---|---|---|---|
| NFR-001 | 성능 | 가상 피드백/비교 카드 계산 API 응답 속도가 사용자 체감 지연을 유발하지 않아야 한다 | 계산 API 평균 응답 500ms 이내 (PM 제안값, 확정 시 조정 가능) |
| NFR-002 | 보안 | 내부 정산 수치(할인율·페이백율·보정계수·커미션율·마진율)는 API 응답·클라이언트 번들 어디에도 노출되지 않아야 한다 | API 응답 페이로드에 결과값(금액·%p)만 포함, 원자료 필드 없음 |
| NFR-003 | 접근성/국제화 | 기능 1·2·3의 모든 유저 노출 문구는 TetherMax 플랫폼이 지원하는 전체 언어로 번역 제공되어야 한다 | 지원 언어 100% 커버리지 (누락 시 폴백 언어 표시) |
| NFR-004 | 가용성 | 프로모션 종료(공통 종료 게이트 발동) 반영은 실시간에 가깝게 적용되어야 한다 | 종료 트리거 발생 후 신규 요청 기준 즉시(수 분 이내) base 로직 반영 |
| NFR-005 | 데이터 정합성 | 기능 1의 UID 페이백 데이터는 일 1회 배치(KST 20:00) 갱신이며, 출금 시점과 배치 시점의 시차를 유저에게 오해를 주지 않는 방식으로 처리해야 한다 | 배치 미반영 구간의 데이터로 계산 시에도 역산 공식이 오류 없이 동작(음수·미계산 시 REQ-006 기준으로 base 폴백) |
| NFR-006 | 플랫폼 호환성 | 기능 1·2는 PC·MO·App(웹뷰) 동일 로직으로 동작해야 하며, 기능 3의 S3~S5는 Web(PC/MO)에서만 적용된다 | 화면변경목록.md 플랫폼 매트릭스와 실 구현이 일치 |
| NFR-007 | 국제화(시간 표시) | 배치·프로모션 판정에 쓰이는 시각(KST 20:00, D+30 등)은 서버 내부 기준이며, 관련 안내 문구를 화면에 노출할 경우 **사용자 기기의 로컬 타임(디바이스 타임존)으로 변환해 표시**해야 한다 | KST 고정 문구 노출 없이, 서로 다른 타임존의 기기에서 각자의 로컬 시각으로 렌더링됨 (2026-07-03 확정) |

## 3. 우선순위 매트릭스 (MoSCoW)

| 우선순위 | 요구사항 ID | 비율 |
|---|---|---|
| Must (필수) | REQ-001~REQ-014, REQ-016~REQ-025, NFR-001~NFR-007 | 약 92% |
| Should (중요) | (해당 없음 — 접기 기능은 확정되어 Must로 편입) | 0% |
| Could (선택) | REQ-015 (복수 UID 내역 보기) | 약 4% |
| Won't (제외, 본 프로젝트 범위 아님) | 프로모션 상시화(OI-07 종결), 캐시백 엔진 이중곱 버그 수정(OI-10, 별도 트랙), 신규 어드민 종료 UI 개발 | 약 4% |
