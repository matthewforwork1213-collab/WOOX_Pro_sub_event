## A-001. 노출 제어 설정 조회

| 항목 | 내용 |
|---|---|
| 설명 | 노출 제어 페이지 진입 시 현재 5개 플래그(on/off) 상태를 조회해 토글 UI에 반영한다 |
| 입력 | 화면 진입(어드민 인증 세션) |
| 처리 | 1) 어드민 권한 확인 2) `GET /api/admin/promo/visibility`로 5개 플래그 조회 3) 각 토글에 현재값 표시 |
| 출력(UI) | 5개 토글(그룹: 출금완료 S1 / 캐시백 프리뷰 S2 / 로그인 3페이지) 각 ON·OFF 상태 |
| 예외 처리 | 조회 실패 시 오류 안내 + 재시도. 권한 없음 → 접근 차단 |
| 관련 API | GET /api/admin/promo/visibility (API-B01) |

## A-002. 노출 제어 설정 저장 (on/off)

| 항목 | 내용 |
|---|---|
| 설명 | 관리자가 개별 토글을 변경하면 서버에 저장하고, 저장 즉시 유저 프론트 신규 요청에 반영되게 한다 |
| 입력 | 토글 변경(항목 키 + 새 값), 어드민 세션 |
| 처리 | 1) 어드민 권한 확인 2) 변경 항목을 포함한 5개 플래그를 `PUT /api/admin/promo/visibility`로 저장 3) 성공 시 "저장됨" 피드백 4) 저장값은 유저 프론트 `GET /api/promo/status`(API-001)가 그대로 반영(장기 캐시 금지, NFR-004) |
| 저장 대상 | `s1Feedback`·`s1Banner`·`s2Compare`·`s2Banner`·`loginBanners` (각 bool). 기본값 전부 ON |
| 규칙 | 각 항목 **독립 저장**(특정 영역만 OFF 가능). 부분 제어 허용. D+30·이벤트 종속 판정 없음 |
| 예외 처리 | 저장 실패 시 토글 롤백 + 오류 안내. 권한 없음 → 차단. 동시 편집 충돌 시 마지막 저장 우선(⚠️ OI-B2 관련) |
| 관련 API | PUT /api/admin/promo/visibility (API-B02) |

## A-003. 변경 감사 로그 기록

| 항목 | 내용 |
|---|---|
| 설명 | 각 플래그 변경 시 변경 이력을 기록한다 |
| 입력 | 저장 이벤트(A-002) |
| 처리 | 저장 시 **변경자·시각·항목·이전값·이후값**을 감사 로그에 적재 |
| 출력 | (선택) 노출 제어 페이지 하단 최근 변경 이력 표시 |
| 예외 처리 | 감사 적재 실패가 설정 저장 자체를 막지 않도록 분리(저장 성공 우선), 실패는 별도 알림 |
| 미결 | ⚠️ OI-B2: 감사 저장소·보존 기간·조회 UI 범위 확정 필요 |
