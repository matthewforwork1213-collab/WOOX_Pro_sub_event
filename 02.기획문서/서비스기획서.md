## 1. Executive Summary

### 1.1 한 줄 요약

TetherMax 플랫폼 내 유저 행동 접점(출금 완료 화면, 캐시백 비교 결과 페이지)에 손실 회피 편향 기반 UI 넛지를 삽입하여, WOOX Pro 신규 온보딩 후 30일간 유저 전환율과 가입률을 집중 극대화한다.

### 1.2 비즈니스 목표

| 항목 | 내용 |
|---|---|
| 서비스명 | WOOX Pro 30일 온보딩 집중 프로모션 |
| 타깃 플랫폼 | TetherMax Web (PC/MO) · App 웹뷰(S1·S2 한정) |
| 핵심 타깃 | TetherMax 내 WOOX Pro 미사용 멀티 거래소 유저 |
| 비즈니스 가치 | WOOX Pro 파트너 레퍼럴 수익 + TetherMax 플랫폼 활성화 지표 개선 |
| 목표 기간 | 기간을 어드민(백오피스)에서 통제 (2026-07-07 C레벨 결정, OI-07 갱신 — D+30 자동 종료 폐지·상시 노출 가능) |

## 2. 시장 현황 및 비즈니스 기회

### 2.1 시장 현황

| 항목 | 수치 | 출처 |
|---|---|---|
| 국내 크립토 활성 유저 | 약 600만~800만 명 | 금감원 실태조사 2025 |
| 멀티 거래소 동시 이용 비율 | 62% | 업비트 리서치 2025 |
| 거래소 선택 기준 1위 (수수료) | 41% | 금감원 실태조사 2025 |
| 수수료 절감 근거 제시 시 30일 내 전환 의향 | 68% | CryptoCompare 2025 |
| 선물/레버리지 유저의 수수료 민감도 | 일반 유저 대비 2.3배 | CryptoCompare 2025 |
| 출금 완료 직후 크로스셀 CTR (토스 벤치마크) | 6~8% | 토스 UX 분석 2024 |
| 구체적 금액 비교 vs 추상적 설명 전환율 차이 | 4.2배 | CryptoCompare 2025 |

### 2.2 비즈니스 기회

- **기회 1**: TetherMax 플랫폼 내 WOOX Pro 미사용 유저의 출금 완료 접점은 하루 수백~수천 건 발생. 이 "높은 심리적 몰입 상태" 순간을 활용하면 기존 이벤트 배너 대비 CTR 20~35배 이상을 기대할 수 있다. (근거: 마켓리서치 2-3)
- **기회 2**: 페이백 미리보기 결과 화면은 유저가 이미 비용 절감에 관심을 보인 상태. "내 조건 기반 맞춤 비교" 시점이므로 전환 저항이 가장 낮은 순간이다. (근거: 마켓리서치 2-4)
- **기회 3**: WOOX Pro의 일반 유저 기준 수수료 경쟁력(메이커 0.00%)은 객관적으로 검증되어 있으나 인지도가 낮다. 구체적 금액 넛지로 인지-행동 갭을 해소할 수 있다. (근거: 마켓리서치 2-1, 2-7)
- **기회 4**: 개인화된 즉시 비교 정보를 제공하는 리베이트 플랫폼은 경쟁사 중 드물어, TetherMax의 차별화 포인트를 WOOX Pro 전환 레버로 직접 활용 가능하다. (근거: 마켓리서치 2-4)

### 2.3 경쟁 환경

| 경쟁사/서비스 | 핵심 기능 | 강점 | 약점 | TetherMax 대비 |
|---|---|---|---|---|
| Coincarp | 거래소 수수료 표 비교 | 데이터 풍부 | 개인화 없음, 전환 유도 없음 | TetherMax 우위 |
| CryptoRebate.io | 리베이트 신청 대행 | 다수 거래소 지원 | UX 복잡, 실시간 미반영 | TetherMax 우위 |
| 국내 경쟁 플랫폼 | 캐시백 집계 | 한국어 지원 | 개인화 계산 미보유 | TetherMax 우위 |
| **TetherMax (현재)** | 페이백 미리보기 | 개인화 계산, 완성도 높은 UX | WOOX Pro 인지도 낮음 | 기준점 |
| **TetherMax (이번 기능)** | 개인화 비교 + 실시간 넛지 | 행동 접점 최적 타이밍 | 30일 한시 운영 | 업계 최초 수준 |

| 항목 | 내용 |

| 불편 사항 | 거래소마다 수수료가 달라 직접 계산하기 번거롭다 |
| 가치 제안 | "지금 내 거래 패턴 기준으로 WOOX Pro가 얼마나 더 유리한지 바로 보여준다" |
| 전환 트리거 | 구체적 금액 비교 제시 ("월 +45 USDT 절약 가능") |

**페르소나 2: 캐시백 탐색형 신규 유저 — "처음 해보는 이수진 (26세)"**

| 항목 | 내용 |
|---|---|
| 나이/직업 | 26세, 대학원생 / 소규모 투자자 |
| 거래 패턴 | 현물 중심, 월 거래량 약 1,000~5,000 USDT |
| 현재 사용 거래소 | 업비트 + 최근 TetherMax 가입 (페이백 관심) |
| 핵심 니즈 | 내가 모르는 더 좋은 거래소가 있는지 비교해보고 싶다 |

## 4. 서비스 전략

### 4.1 핵심 가치 제안 (Value Proposition)

> "거래 완료 직후, 내 조건 기반으로 계산된 실제 손실 금액을 보여줌으로써 — 유저가 스스로 WOOX Pro로 이동하도록 설계한다."

- **유저에게**: 복잡한 수수료 비교 없이, 내가 이미 진행한 거래 기준으로 즉시 손익 판단 가능
- **TetherMax에게**: 파트너 레퍼럴 전환율 극대화 + 플랫폼 재방문 유도
- **WOOX Pro에게**: 한국 유저 인지도 확보 + KYC 완료 신규 유저 유입

### 4.2 기능 전략

| 단계 | 기능 | 우선순위 | 설명 |
|---|---|---|---|
| MVP | 기능 1 — 출금 완료 후 프로모션 분기 노출 | P0 | WOOX Pro 미포함 출금 시 가상 피드백 카피 노출. 기존 광고 영역 활용으로 개발 공수 최소화 |
| MVP | 기능 2 — 캐시백 프리뷰 결과 인라인 비교 카드 | P0 | 타 거래소 선택 시 결과 카드 아래 WOOX Pro 비교 카드 인라인 삽입(팝업·딤 없음). 개인화 금액 표시 핵심 (OI-04 확정) |
| MVP | 기능 3 — 트레블룰 연동 신뢰 배너 | P0 | WOOX Pro × 빗썸 트레블룰 연동을 5개 위치에 배너로 노출(**클릭 시 WOOX Pro 상세 이동**, 기능 3 개정 2026-07-07). `[빗썸]⇄[WOOX Pro]` 로고(디자인 하드코딩, 사이 ⇄) + "트레블룰 연동" 텍스트, 로고 앞·문구 뒤 고정, 절제형. 노출은 백오피스 트레블룰 배너 on/off로 제어(2026-07-07: D+30·2개 WOOX 이벤트 게이트 폐지). ⚠️ 정적 배너 전제였던 OI-09 법무 리스크 수용 재확인 필요 |
| MVP | 역효과 방지 로직 | P0 | 타 거래소가 실제로 유리한 케이스 필터링 — 비교 카드 미삽입 |
| MVP | 노출 제어 (백오피스 on/off) | P0 | 영역별 on/off 5개로 **독립 제어**(부분 제어 허용). 신규 백오피스 페이지 + 어드민 API(B01/B02). 2026-07-07: D+30·2개 WOOX 이벤트 게이트 폐지, 기간도 어드민 통제 |
| MVP | 비교 카드 접기(collapse) | P1 | 인라인 카드 접기 제공 확정 (OI-05, Must). 강제 닫기(X) 없음 |
| 확장 | 복수 UID "내역 보기" | P2 | 복수 거래소 합산 총액에서 UID별 절약 내역 펼치기 (선택적 디자인 옵션) |

### 4.3 차별화 전략

- **차별화 1 — 개인화 즉시 비교**: 유저가 입력한 조건(거래소, 자산 규모, 레버리지, 거래 빈도) 기반으로 계산된 실제 금액 제시. 범용 광고 배너와 근본적으로 다름.
- **차별화 2 — 행동 접점 타이밍 최적화**: 출금 완료 직후(기능 1), 비교 결과 진입 직후(기능 2)라는 심리적 몰입도가 극도로 높은 순간에만 노출.
- **차별화 3 — 신뢰 기반 강압 없는 설계**: 접기(collapse) 제공(강제 닫기 X 없음), 비교 기준 하단 명시, 역효과 케이스 미노출으로 유저 신뢰를 유지하면서 전환을 유도.

## 6. 수익 모델

### 6.1 채널과 파트너

| 채널 | 역할 | 수익 연결 |
|---|---|---|
| TetherMax 플랫폼 | 유저 트래픽 보유, 넛지 기능 운영 | 레퍼럴 전환 수익 수취 |

| 유저 | WOOX Pro 가입 및 거래 | 수수료 절감 혜택 수령 |

### 6.2 수익 구조

이번 기능은 TetherMax 플랫폼의 기존 WOOX Pro 파트너십 레퍼럴 수익 구조를 기반으로 한다. 별도 유료화 또는 가격 정책은 없으며, 성과 기반 구조로 운영한다.

| 수수료 데이터 갱신 | 거래소별 수수료율의 주기적 업데이트 (초기: 수동, 이후: 자동화 검토) | 낮~중 |
