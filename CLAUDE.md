### 도메인 컨텍스트

**서비스 개요**: TetherMax 플랫폼 내 유저 행동 접점(출금 완료 화면, 캐시백 프리뷰 결과 페이지)에 WOOX Pro 전환 유도 넛지를 추가하는 프로모션 기능.

- **기능 1 — 출금 완료 후 WOOX Pro 프로모션 분기 노출**: WOOX Pro 미포함 출금 시, 기존 이벤트 광고 대신 "WOOX Pro 사용 시 수수료를 얼마나 더 아낄 수 있었는지"(토탈 세이빙 기준) 가상 피드백 노출.
- **기능 2 — 캐시백 프리뷰 결과 페이지 인라인 비교 카드**: 페이백 미리보기에서 타 거래소 선택 시, 결과 카드 아래에 WOOX Pro 비교 카드를 인라인 삽입(팝업·딤 없음). 기존 "딤+강제 모달" 방식에서 전환(OI-04).
- **기능 3 — WOOX Pro × 빗썸 트레블룰 연동 배너**: WOOX Pro가 빗썸과 트레블룰을 연동한 거래소임을 알리는 배너를 5개 위치(프리뷰 결과 하단·출금 결과 카드 하단·모바일 로그인 전·PC 로그인 페이지 상단·로그인 후 마이페이지 회원 ID 하단)에 노출. **클릭 시 WOOX Pro 거래소 상세로 이동**(CTA URL=어드민 온보딩 등록 값 OI-06. 기능 3 개정 2026-07-07 — 기존 "정적·클릭 없음"에서 변경. ⚠️ 정적 정보 배너 전제였던 OI-09 법무 리스크 수용 재확인 필요). 프로모션 한정(공통 종료 게이트: 2개 WOOX Pro 이벤트 종료 버튼 OR + D+30). 구성은 `[빗썸 로고]⇄[WOOX Pro 로고]`(디자인 하드코딩, 로고 사이 좌우 화살표 ⇄) + "트레블룰 연동"(다국어 텍스트), 로고 앞·문구 뒤 순서 고정, 절제형 뱃지.

### 도메인 용어

| 용어 | 설명 |
|---|---|
| 넛지 (Nudge) | 유저를 강제하지 않고 특정 행동을 유도하는 UI/UX 설계 기법 |
| 손실 회피 편향 | 동일한 금액의 이익보다 손실을 더 크게 인식하는 심리 편향 |
| 가상 피드백 | "WOOX Pro를 사용했더라면 수수료를 얼마나 더 아낄 수 있었는지"(토탈 세이빙 기준) 계산된 반사실 문구 |
| 딤(Dim) 처리 | 배경 화면을 반투명하게 흐리게 만드는 UI 처리 |
| 페이백 미리보기 | 유저가 거래소 조건을 입력하면 예상 캐시백을 미리 계산해 보여주는 기능 |
| 프로모션 분기 | WOOX Pro 포함/미포함 여부에 따라 노출 콘텐츠를 다르게 처리하는 로직 |
| CTA 버튼 | Call To Action — 유저를 WOOX Pro 가입/상세 페이지로 이동시키는 버튼 |
| 인라인 비교 카드 | 캐시백 프리뷰 결과 페이지에서 결과 카드 아래에 삽입되는 WOOX Pro 비교 카드 (팝업·딤 없는 비강제 노출) |
| 트레블룰(Travel Rule) | 가상자산 이전 시 송수신자 정보를 거래소 간 공유하도록 하는 규제. 거래소 간 "트레블룰 연동"은 상호 자금 이전 지원·규제 준수 신호로 신뢰 요소가 됨 |
| 트레블룰 배너 | WOOX Pro × 빗썸 트레블룰 연동을 알리는 신뢰 배너(기능 3). 로고=이미지(디자인 하드코딩, 로고 사이 ⇄), 문구="트레블룰 연동"(다국어 텍스트), **클릭 시 WOOX Pro 상세 이동**(기능 3 개정 2026-07-07, ⚠️ OI-09 법무 재확인) |
| 커미션율 | 파트너 계정(테더맥스)이 거래소로부터 받는 수수료 분배 비율 (WOOX Pro 90%, 내부 전용). **유저 화면 계산에는 사용하지 않음** |
| 페이백율 | 유저가 실제 낸 수수료 대비 캐시백으로 환급하는 비율 (WOOX Pro 80%, 내부 전용) |
| 할인율(수수료 할인율) | 거래소 원 수수료 대비 유저에게 즉시 할인해주는 비율 (WOOX Pro 0%, 내부/백오피스) |
| 토탈 세이빙(율) | 할인 + 페이백을 합친 유저 실질 절감. 명목: `1−(1−할인율)×(1−페이백율)`(기능 1). 프리뷰: `1−(1−할인율)×(1−페이백율×0.7)`(기능 2, 보정계수 0.7 고정). WOOX Pro는 할인 0이라 명목값은 페이백율과 동일(80%). 내부 전용, 결과값만 노출 |
| 마진율 | 커미션 중 테더맥스가 갖는 몫. 백오피스는 페이백율·마진율만 저장(커미션율 미입력). 내부 전용 |
| 보정계수(Correction Factor) | 캐시백 프리뷰(기능 2)의 Monthly Estimated Cashback 산식에 곱해지는 계수. **0.7 고정 상수**(전 거래소·전 상품 동일, 하드코딩 가능). 실제 발생 데이터(기능 1)에는 적용 안 함, 프리뷰 예측치(기능 2)에만 적용 |

## 핵심 비즈니스 규칙 (산출물 작성 시 반드시 준수)

1. **역효과 방지 (토탈 세이빙 기준)**: 노출 여부 판정은 **토탈 세이빙율**로 통일. 타 거래소 토탈세이빙율이 WOOX Pro와 **같거나 높으면** 광고·비교 카드 미노출(WOOX Pro가 엄격히 클 때만 노출). 표시 수치도 토탈 세이빙 기준으로 확정(OI-08)되어 판정·표시가 동일 산식.
2. **기준 명시 의무**: 모든 수수료 비교는 '일반 유저/기본 등급' 기준임을 비교 카드 하단에 소문자로 명시.
3. **인라인 비강제 노출 (기능 2)**: 팝업·딤 없이 결과 페이지에 인라인 비교 카드로 노출. 조건 충족 시 상시 노출하며, 강제 닫기(X) 없이 **접기(collapse) 제공**. (기존 "세션당 1회 팝업" 규칙 폐기, OI-04·OI-05)
4. **WOOX Pro 선택 시 비교 카드 미노출**: 유저가 이미 WOOX Pro를 선택한 경우 비교 카드를 삽입하지 않음.
5. **CTA 브랜드 컬러 준수**: Primary CTA 버튼은 TetherMax 브랜드 컬러 적용.
6. **내부 수치 비노출**: 커미션율(90%)·페이백율(80%)·할인율·마진율 원자료 및 예산 등 내부 수치는 유저 화면에 절대 표시 금지. 유저에게는 계산 결과값(절약 금액 USDT, %p)만 노출. (예산 등 대외 공개 카피는 비전 프레이밍 사용)
7. **계산 기준**: 유저 화면 절약액 계산은 **토탈 세이빙율** 기준으로 산정하며 **커미션율은 사용하지 않는다.** 출금 화면(기능 1)은 실제 페이백 금액에서 기준 수수료를 역산한다. 표시 방식은 **토탈 세이빙 기준으로 확정**(OI-08, 2026-07-03 — 페이백률 기준은 폐기). (PRD §2-A-1)
8. **%p 정의**: 표시 %p = **명목 토탈세이빙율(WOOX Pro) − 명목 토탈세이빙율(현재 거래소)**, 보정계수(0.7) 미반영. USDT 절약 금액은 기능별로 다른 산식(기능1=실측 역산, 기능2=보정계수 반영)을 쓰지만 %p는 항상 이 명목 산식으로 통일 (기능 1·2 공통, 2026-07-03 확정, PRD §2-A-1)
9. **시간 표시 규칙**: 배치 처리(UID 페이백 갱신)·프로모션 활성 판정(D+30)은 서버 기준 **KST 절대시각**으로 처리한다. 다만 이 시각을 **유저 화면에 노출할 경우 사용자 기기의 로컬 타임(디바이스 타임존)으로 변환해 표시**한다 — KST 고정 문구로 노출 금지. (2026-07-03 확정)

## Gate-Check Rules (V0.43)

산출물 완료 시 아래 5단계 프로세스를 반드시 준수합니다.

1. **사전 확인**: 산출물 작성 전 `.progress.md`를 읽고 전제조건(상태=완료) 확인
2. **미충족 시**: 사용자에게 미충족 항목을 안내하고, 선행 산출물 작성을 제안
3. **완료 처리** (산출물 작성 후 순서대로 실행):
   - (a) `.progress.md`의 상태를 "완료"로, 완료일을 기록
   - (b) 가능하다면 NotebookLM에 해당 산출물을 소스로 등록
   - (c) 등록 실패 시 `.progress.md` 비고란에 `[NLM 미등록]` 기록 후 다음 작업 계속
4. **실체 검증**: 테스트결과보고서(#16)는 `tests/` 파일 존재 확인, 완료보고서(#17)는 15개 이상 산출물 완료 확인
5. **강제 스킵**: 사용자가 "gate-check 무시하고"라고 명시할 경우 경고 후 진행, `.progress.md`에 `[SKIP]` 태그 기록

### Gate-Check 품질 기준

| 항목 | 기준 |
|---|---|
| 한국어 작성 | 모든 산출물은 한국어 (영문 용어는 괄호로 설명) |
| 이모지 미사용 | 이모지 금지 |
| 표 활용 | 복잡한 정보는 표로 구조화 |
| OI 태깅 | 미결 사항은 OI-## 형식으로 표시 |

## Git Conventions

### 커밋 메시지 형식

```
[영역] 간단한 설명

상세 내용 (필요 시)
- 항목 1
- 항목 2

Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>
```

**영역 (Scope):**
- `docs:` — 산출물 문서 추가/수정
- `feat:` — 기능 개발 (코드)
- `fix:` — 버그 수정
- `chore:` — 설정, 폴더 구조
- `test:` — 테스트 코드

### 브랜치 전략

| 브랜치 | 용도 | 생성 규칙 |
|---|---|---|
| `main` | 프로덕션 | PR 병합으로만 업데이트 |
| `develop` | 개발 통합 브랜치 | PR 리뷰 후 병합 |
| `feature/*` | 기능 개발 | `develop`에서 생성 |
| `docs/*` | 산출물 작성 | `develop`에서 생성 |
| `fix/*` | 버그 수정 | `develop`에서 생성 |
